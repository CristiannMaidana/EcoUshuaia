# Roadmap tecnico

## Estado

Actualmente el repositorio no incluye un roadmap tecnico formal versionado como parte del codigo.

## Proposito de este documento

Este archivo deja reservado el espacio para documentar, cuando corresponda:

- prioridades tecnicas
- mejoras estructurales
- hitos de producto
- decisiones de evolucion por plataforma

## Sugerencia de uso

Si en el futuro se define un roadmap, conviene registrar al menos:

- objetivo
- alcance
- fecha o iteracion
- impacto esperado
- modulos afectados

## Contexto actual

Por la estructura del repositorio, las areas que concentran mas complejidad tecnica hoy son:

- `features/map`
- integracion nativa iOS en `ios/Runner`
- composicion de providers en `lib/app/di/di.dart`

Este documento puede evolucionar junto con la planificacion tecnica del proyecto.
